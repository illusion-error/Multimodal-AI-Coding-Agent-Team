二、本次排查发现的问题清单

问题 1：backend 目录不完整

现状：
   backend/ 下面只有 Dockerfile。

影响：
   Dockerfile 中启动命令是：
   uvicorn main:app --host 0.0.0.0 --port 8000

   但 backend/main.py 不存在，因此后端容器启动一定失败。

建议：
   新增 backend/main.py，提供 FastAPI app 和全部 /api 接口。


问题 2：backend/Dockerfile 的依赖文件路径不对

现状：
   backend/Dockerfile 中写的是：
   COPY requirements.txt .

   但 Docker build 上下文是 ./backend，
   backend/requirements.txt 当前不存在。

影响：
   docker compose build backend 时会找不到 requirements.txt。

建议二选一：
   方案 A：在 backend/ 下新增 requirements.txt。
   方案 B：修改 docker-compose.yml 的 backend build context 为项目根目录，并指定 dockerfile。

推荐方案 A，最简单：
   backend/requirements.txt 至少包含：
   fastapi
   uvicorn[standard]
   python-multipart
   pillow


问题 3：根目录 requirements.txt 不适合直接给后端容器用

现状：
   根目录 requirements.txt 仍偏向原 Streamlit / E2B / Gemini 项目：
   streamlit
   e2b-code-interpreter
   agno>=2.2.10
   Pillow
   openai
   google-genai

影响：
   缺少 fastapi、uvicorn、python-multipart。
   后端接口服务无法直接运行。

建议：
   保留根目录 requirements.txt 用于 Streamlit 单文件演示；
   backend/requirements.txt 单独用于后端接口。


问题 4：前端接口已经写死 127.0.0.1:8000

现状：
   frontend/src/api/task.js：
   const BASE_URL = 'http://127.0.0.1:8000'

影响：
   本地开发可以用，但部署到服务器或 Docker 环境时不灵活。

建议：
   后端先按 8000 端口实现，保证前端能跑通。
   后续前端可改成 Vite 环境变量：
   VITE_API_BASE_URL=http://127.0.0.1:8000


问题 5：成员 C 的 AgentStep 字段与前端组件字段不一致

成员 C 当前字段：
   name
   role
   status
   input_summary
   output_summary
   duration_ms
   error

前端 AgentSteps.vue 当前读取：
   agent_name 或 step_name
   status
   duration_ms
   input
   output
   error

影响：
   如果后端直接返回 AgentStep 原始对象，前端会显示“步骤 1、步骤 2”，输入输出内容也显示为空。

后端必须转换：
   name -> agent_name
   role -> role
   input_summary -> input
   output_summary -> output


问题 6：成员 C 的 execution_report 是字符串，前端需要对象

成员 C 当前返回：
   execution_report: str

前端 HomeView.vue 读取：
   result.execution_report.exit_code
   result.execution_report.stdout
   result.execution_report.stderr

影响：
   如果后端直接返回字符串，前端执行结果页会显示不正常。

后端必须统一转换为：
   execution_report: {
     "status": "success | failed | timeout | blocked",
     "exit_code": 0,
     "stdout": "...",
     "stderr": "...",
     "raw": "原始执行报告"
   }


问题 7：成员 D 的 sandbox/code_runner.py 与成员 C 的 run_python_code 暂未打通

成员 D 当前提供：
   execute_code_safely(code_str, timeout_seconds=5) -> dict

成员 C 当前内部使用：
   run_python_code(code, timeout_seconds) -> str

建议：

   如果时间允许，再把成员 C 的 run_python_code 替换为调用 sandbox.execute_code_safely。


问题 8：测试生成 Agent 已有，但结构化测试用例没有直接放进 AgentResult

成员 C 当前有：

接口 2：提交文本题目

POST /api/tasks/text

请求体：
{
  "problem_text": "两数之和：给定 nums 和 target，返回两个数的下标。"
}

推荐处理流程：
1. 生成 task_id。
2. 先保存任务状态为 running。
3. 调用成员 C：
   result = solve_problem(config=config, text_problem=problem_text)
4. 把 AgentResult 转成前端需要的 JSON。
5. 保存任务状态为 completed 或 failed。
6. 返回 task_id。

返回：
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "task_20260617_001",
    "status": "running"
  }
}

说明：
   前端会拿 task_id 调用 pollTaskStatus。
   第一阶段如果不会写异步任务，也可以同步执行完成后再返回 task_id，
   只要 GET /api/tasks/{task_id} 能返回 completed 即可。


接口 3：提交图片题目

POST /api/tasks/image

请求类型：
   multipart/form-data

字段：
   image: 上传图片文件
   supplement: 补充说明，可为空

后端调用：
   image_bytes = await image.read()
   result = solve_problem(
       config=config,
       text_problem=supplement,
       image_bytes=image_bytes,
       image_mime=image.content_type or "image/png",

返回：

  "message": "success",
  "data": {
    "task_id": "task_20260617_002",
    "status": "running"
  }
}

九、必须写的两个转换函数

1. Agent 步骤转换

def normalize_steps(agent_steps):
    return [
        {
            "step_order": index,
            "agent_name": step.name,
            "step_name": step.name,
            "role": step.role,
            "status": step.status,
            "input": step.input_summary,
            "output": step.output_summary,
            "duration_ms": step.duration_ms,
            "error": step.error,
        }
        for index, step in enumerate(agent_steps, start=1)
    ]


2. 执行报告转换

def normalize_execution_report(raw_report):
    raw = raw_report or ""
    exit_code = None
    stdout = ""
    stderr = ""

    if "退出码：0" in raw or "退出码:0" in raw:
        exit_code = 0
        status = "success"
    elif "执行超时" in raw:
        status = "timeout"
    elif "安全检查" in raw:
        status = "blocked"
    else:
        status = "failed"

    if "标准输出：" in raw:
        stdout = raw.split("标准输出：", 1)[1].split("错误输出：", 1)[0].strip()
    if "错误输出：" in raw:
        stderr = raw.split("错误输出：", 1)[1].strip()

    return {
        "status": status,
        "exit_code": exit_code,
        "stdout": stdout,
        "stderr": stderr,
        "raw": raw,
    }
