  "stdout": "",

  "exit_code": 0,

4.4 不使用子字符串安全检查

当前 read 会误伤：
- breadth
- threading

且字符串拼接可绕过。

改用 ast：

- 检查 ImportFrom。
- 检查 Call。
- 检查 Attribute。

但必须明确：
AST 过滤不是安全边界。

4.5 第一阶段执行器最低要求

- 使用 sys.executable。
- 使用 Python -I 隔离模式。
- 使用 -B 禁止写 pyc。
- 临时目录。
- 最小环境变量。
- 超时终止。
- 捕获 stdout/stderr。
- 限制输出长度。
- 不记录 API Key。

4.6 第二阶段真正沙盒

使用 Docker Sandbox 或 E2B。

Docker 运行限制：
- --network none
- --read-only
- --memory
- --cpus
- --pids-limit
- --security-opt no-new-privileges
- 临时 tmpfs
- 超时后删除容器
