三、P0：成员 B 第一阶段后端必须修改

1. 接入真实 Agent，删除永久模拟模式

- 正确调用：

- API 没有密钥时可以使用 C 的离线兜底，但不能使用后端自造的固定模拟结果。

2. 不要维护两份 Agent 主程序

当前问题：
- 根目录有 ai_coding_agent_bailian.py。
- backend 下又复制了一份相同文件。
- C 修改根目录版本后，后端副本可能不同步。

修改要求：
- 只保留根目录 ai_coding_agent_bailian.py 作为唯一来源。
- backend/main.py 从根目录模块导入。
- Docker 构建时同时复制根目录 Agent 文件和 backend 目录。

3. 修正 AgentResult 适配方式

当前问题：
- solve_problem 返回 AgentResult 数据类，不是 dict。
- 后端使用 result.get(...)，接入真实 Agent 后会立即报错。

正确字段映射：
- result.problem -> problem
- result.solution_markdown -> solution_markdown
- result.code -> code
- result.execution_report -> 执行结果
- result.project_document -> 完整报告
- result.metrics -> total_ms、模型、运行状态等
- result.agent_steps -> Agent 步骤表
- result.retrieved_templates -> rag_hits
- result.test_plan -> 测试计划
- result.repair_attempts -> 修复日志
- result.api_used -> api_call
- result.fallback_used -> fallback_used

4. 修正数据库字典读取错误

当前问题：
- database.py 的 list_all_tasks、get_steps_by_task、get_tests_by_task 返回字典。
- main.py 仍按旧元组方式拆包。
- 实测历史列表返回：
  task_id = "task_id"
  status = "status"
  created_at = "created_at"

修改要求：
- 全部改成 row["task_id"]、row["status"] 等字典读取方式。
- 步骤和测试接口同样按字段名读取。

5. 禁止使用 INSERT OR REPLACE 更新任务

当前问题：
- 任务开始时插入 running 记录。
- 执行过程中插入 Agent 步骤和测试用例。
- 完成时 INSERT OR REPLACE 会先删除旧任务。
- 外键级联删除会把刚写入的步骤和测试一起删除。
- 实测步骤接口和测试接口最终均为空数组。

修改要求：
- 使用 INSERT ... ON CONFLICT(task_id) DO UPDATE。
- 或分别实现 create_task 和 update_task。
- 更新状态时必须保留原 created_at。

6. 修正图片上传字段

当前问题：
- 前端 FormData 使用字段 image。
- 后端参数名是 file。
- 实测前端格式请求返回 422。

修改要求：
- 后端改为 image: UploadFile = File(...)。
- 校验 MIME 类型和文件大小。
- 把真实 image_bytes、content_type 和补充文本传给 solve_problem。
- 不得只把文件名和大小拼成“模拟题目”。

7. 修正历史列表返回字段

前端历史页面需要：
- task_id
- status
- problem
- created_at
- total_ms

当前后端只查询 task_id、status、created_at。

修改要求：
- 从 data JSON 中取出 problem 和 total_ms。
- 返回前端所需的完整摘要。

8. 修正 Agent 步骤返回字段

建议统一返回：
- agent_name
- role
- status
- input
- output
- duration_ms
- error

当前后端返回 name 和 duration，而 AgentTimeline 读取 step_name/agent_name 和 duration_ms。

9. 修正报告下载接口

当前问题：
- 后端返回 JSON：{code, message, data: markdown}。
- 前端把整个 response.data 当 Markdown 文件。
- 下载后可能得到 JSON 或 [object Object]。
- 后端生成的 Markdown 代码块没有闭合。

推荐方案：
- 后端直接返回 text/markdown 的 Response。
- 优先返回 result.project_document。
- 文件名通过 Content-Disposition 指定。

10. 修正重新执行协议

当前问题：
- 后端重新执行时创建新 task_id。
- 前端仍轮询旧 task_id。
- 页面会显示旧结果，无法看到新任务结果。

二选一：
- 方案 A：后端复用原 task_id，清理旧步骤后重新执行。
- 方案 B：后端返回新 task_id，前端立即更新 taskId 并轮询新 ID。

推荐方案 B，保留原任务和重跑任务之间的 original_task_id 关系。

11. 正确保存测试用例和修复日志

当前问题：
- execution_logs 表已建立，但 main.py 没有写入。
- C 的 repair_attempts 没有保存。
- C 内部生成了 test_cases，但 AgentResult 当前没有返回该字段。

处理方式：
- C 给 AgentResult 增加 test_cases 字段。
- B 将 test_cases 写入 test_cases 表。
- B 将 repair_attempts 写入 execution_logs 表。

12. 规范 execution_report

当前问题：
- C 当前返回字符串。
- 前端和数据库指标按对象读取 exit_code、stdout、stderr。

建议后端统一转换为：
{
  "exit_code": 0,
  "stdout": "...",
  "stderr": "",
  "raw": "原始执行报告"
}

13. 修正后端启动编码问题

当前问题：
- Windows GBK 环境下，startup_event 中的 emoji 会导致 UnicodeEncodeError。
- 实测后端启动阶段直接退出。

修改要求：
- 后端日志不使用 emoji。
- 或统一设置 UTF-8 日志输出，但代码本身仍建议使用普通文字日志。

14. 补充 python-multipart 依赖

图片上传接口依赖 python-multipart，但 requirements.txt 和 backend/requirements.txt 都没有声明。

必须加入：
python-multipart

15. 修正包导入与 Docker 启动路径

当前问题：
- backend/main.py 使用 from database import ...。
- 从项目根目录执行 import backend.main 会报 ModuleNotFoundError。
- Docker 构建上下文是 ./backend，但 CMD 使用 backend.main:app。
- 镜像内只有 /app/main.py，没有 /app/backend/main.py。

建议结构：
- docker-compose 的 backend build context 改为项目根目录。
- dockerfile 指向 backend/Dockerfile。
- Docker 中复制：
  ai_coding_agent_bailian.py
  backend/
  requirements.txt
- main.py 使用 from backend.database import ... 或相对导入。
- CMD 使用 uvicorn backend.main:app。
